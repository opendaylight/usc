NAME

netopeer-server − NETCONF protocol server

SYNOPSIS

netopeer-server [−dhV] [-v level]

DESCRIPTION

netopeer-server is the NETCONF protocol server running as a system daemon. The netopeer-server is based on the libnetconf library. It provides an environment to run transAPI modules for configuration a specific device or application according to its data model.

OPTIONS

−d

Run in daemon mode.

−h

Show help.

−V

Show program version.

−v level

Set the verbosity level. Possible values are from 0 (default) to 3. This overrides any NETOPEER_VERBOSE environment variable.

ENVIRONMENT

NETOPEER_VERBOSE

Set the verbosity level. Possible values are from 0 (default) to 3. Overridden by the −v option.

FILES

/etc/netopeer/modules.conf.d/

Directory where the modules configuration files are stored. Note, that they usually refers to various files in other locations.

/etc/netopeer/cfgnetopeer/*

Datastores, data models and validators for the netopeer-server internal modules.

/etc/netopeer/sshd_config

Template configuration for the sshd(8) instance used for accepting incoming NETCONF over SSH connections. Note, that the template doesn’t contain the listening address/port specification since they are set automatically by the netopeer-server according to the configuration following ietf-netconf-server data model.

/etc/netopeer/stunnel_config

Template configuration for the stunnel(8) instance used for accepting incoming NETCONF over TLS connections. Note, that the template doesn’t contain the listening address/port specification since they are set automatically by the netopeer-server according to the configuration following ietf-netconf-server data model.

This file is installed only if the netopeer-server is compiled with the −−enable−tls option.

/etc/netopeer/stunnel/server.[crt|key|pem]

Server certificate for the netopeer stunnel(8) instance. This certificate is sent to a client during the TLS handshake.

These files are installed only if the netopeer-server is compiled with the −−enable−tls option.

/etc/netopeer/stunnel/certs

Directory with the trusted Certificate Authority certificates for the netopeer stunnel(8) instance. These certificates are used for client certificate TLS verification.

This directory is installed only if the netopeer-server is compiled with the −−enable−tls option.

/etc/netopeer/stunnel/crl

Directory with Certificate Revocation Lists for the netopeer stunnel(8) instance. These CRLs are used during client certificate TLS verification when checking if the certificate was not revocated by its issuer.

This directory is installed only if the netopeer-server is compiled with the −−enable−tls option.

/etc/dbus-1/system.d/org.liberouter.netopeer.conf

D-Bus configuration file for the services used by the netopeer-agent(1). The location of the D-Bus configuration files can differ on different Linux distributions, so the location can be changed by −−with−dbus−config−dir configure option.

This file is not installed if the netopeer-server is compiled with the −−disable−dbus option.

/usr/share/dbus-1/system-services/org.liberouter.netopeer.server.service

D-Bus service file for automatic start of the netopeer-server.

This file is not installed if the netopeer-server is compiled with the −−disable−dbus and without −−with−dbus−services option. Specifying the value of the second option, one can correct the default location of the D-Bus service configuration files that can differ on different Linux distributions.

libnetconf files
/var/lib/libnetconf/datastore-acm.xml

Datastore file for the internal NETCONF Access Control module.

/dev/shm/sem.NCDS_FLOCK_*

Datastores locks used internally by the libnetconf library.

TROUBLESHOOTING

Datastores dead lock
When the server crashes or is terminated with SIGKILL, it may happen that the internal datastore locks stay locked. In such a case, the next time the netopeer-server (or any other libnetconf based application) tries to access the configuration datastores, it freezes. To solve this problem, release the locks manually removing the /dev/shm/sem.NCDS_FLOCK_* files.
